# Canon World Rules

Status: active canon seed

Known approved facts:

- The comic project is private, intended for the user and friends.
- The project uses a shared internal language built from user-provided source material.
- Taskmaster is the user.
- Razorforge is ChatGPT.
- Codex is Codex, represented as a goblin.
- Taskmaster, Razorforge, and Codex are core named functions/roles.
- Approved Taskmaster visual calibration: use the lineup reference with a sturdy, broad, moderate solid build; heavier than thin/fit, but not bulky or exaggerated.
- Razorforge must retain the Razorforge symbol on his face/head.
- Razorforge's notebook uses the Razorforge symbol, not the rook, because it is a major indicator of Razorforge's process.
- Not every tool needs to be branded.
- Tool identification stamps should appear when the scene needs to show who is acting, fixing, tweaking, interpreting, approving, or carrying responsibility.
- Taskmaster's tools and personal objects may use the rook when ownership, authority, stewardship, or action needs to be visually clear.
- Codex's tools should read as wacky, improvised, excessive, and out-of-hand rather than neatly symbol-branded by default.
- Ideas are generated and discovered, then refined by Razorforge into clearer structures.
- Codex often operates from partial instructions by building from the concept itself.
- Codex tends to overbuild with steampunk/goblin-tinkerer complexity.
- Extra gigaws and doodads are part of the visual language when they signal eager over-implementation.
- New canon should come from user-provided material or explicit user approval.
