# Character Creation Guidance

Status: working project guidance

Date: 2026-07-04

Purpose:

Define how Comics / Around Here should treat a character candidate differently from a pure symbol candidate.

## Standard Term

Use `character snapshot`.

A character snapshot is a compact visual and behavioral reference for a comic character.

It is not an RPG character sheet.

It does not include stats, abilities, game mechanics, inventory, or role-playing build information.

It should capture:

- how the character looks
- how the character acts
- what must stay visually stable
- what expressions or poses are natural
- what props or body logic matter
- what drift risks to avoid

When Taskmaster asks for a character snapshot, create a comic-character reference, not a role-playing artifact.

## Core Distinction

A symbol is primarily a repeatable visual meaning.

A character is a repeatable actor.

Characters may carry symbolic meaning, but they must also be able to:

- act
- react
- hold props
- show emotion
- make choices
- occupy scenes from more than one angle
- remain recognizable across poses and expressions

Do not weaken a character into a flat icon merely because the character has symbolic function.

## Character Candidate Standards

When Taskmaster asks to create or refine a character, preserve:

- role
- jurisdiction or narrative function
- personality / acting range
- required visual anchors
- optional props
- body logic
- expression range
- use limits
- drift risks

Character guidance should usually include:

- status
- source
- role
- core snapshot
- what the character does
- what the character does not do
- visual anchors
- acting notes
- prop handling
- expression notes
- appropriate scenes
- avoid list
- current candidate images

## Visual Standards For Characters

Characters should stay within the project visual language:

- black-and-white by default
- newspaper-comic / comic-strip line art
- readable silhouette
- not realistic or hyper-realistic unless explicitly requested
- not color unless Taskmaster explicitly says `full color`
- aligned with source notes and approved corrections

But characters may be more developed than symbols:

- more facial expression
- more pose variation
- more body acting
- more personality in posture
- more recurring props or wardrobe
- more detail when it helps acting and recognition

The standard is not:

> reduce until only an icon remains

The standard is:

> keep the character readable, repeatable, and style-consistent while preserving acting ability.

## Symbol-Loaded Characters

Some characters are also symbols.

For these, separate:

- symbolic load
- character role
- acting range

Example:

Chief Inspector:

- symbolic load: category claim under inspection
- character role: investigator of category fraud
- acting range: focused, skeptical, frustrated, dry, professionally irritated, quietly satisfied when a classification resolves

Do not let the symbolic load erase the acting range.

## Prop And Body Logic

If a character uses tools, the body must be designed to use them.

Examples:

- claws must visibly grip a magnifying glass
- hands must actually hold notebooks or tools
- goggles should sit consistently on a head or forehead
- mechanical props should connect to the body or scene

Do not place props near characters as decoration when the scene implies active use.

## Candidate Versus Canon

Character candidates are not canon until Taskmaster approves them.

Candidate images may explore:

- face
- posture
- props
- expression range
- silhouette
- outfit
- role-specific staging

Approved character guidance should be moved or copied into `Source/Characters/` only after Taskmaster approval.

Until then, keep candidate material in `Working/` or `Images/Generated/` and mark it clearly.

## Review Question

When evaluating a character image, ask:

```text
Can this figure act in more than one scene while still reading as itself?
```

If yes, evaluate it as a character.

If no, evaluate it as a symbol or one-off illustration.
