==================================================
CORE PHILOSOPHY
==================================================

The comic is idea-driven.

Characters are symbolic tools.

Objects, systems, symbols, failures,
patterns, relationships, discoveries,
and recurring motifs are equally important.

Preferred hierarchy:

Visual Story
↓
Symbol
↓
Character Action
↓
Dialogue

Dialogue should reinforce the idea.

Dialogue should not carry the idea.

==================================================
THE TEST
==================================================

Remove all dialogue.

Question:

Can the reader still understand
what is happening?

If yes:
Comic succeeds.

If no:
More symbolism.
Less exposition.

==================================================
VISUAL STORYTELLING RULE
==================================================

Highest Priority Rule.

Object > Dialogue

Symbol > Explanation

Visual Gag > Exposition

The reader should understand the joke
before reading the words.

==================================================
CHARACTER PURPOSES
==================================================

TASKMASTER

Represents:

- Stewardship
- Responsibility
- Continuity
- Prioritization
- Ownership
- Execution

Taskmaster carries the thing forward.

Taskmaster is not bureaucracy.

Taskmaster is ownership.

--------------------------------------------------

RAZORFORGE

Represents:

- Interpretation
- Pattern Recognition
- Analysis
- Synthesis
- Structural Thinking

Purpose:

Transform information
into understanding.

RazorForge identifies leverage.

RazorForge reveals relationships.

RazorForge rarely creates.

--------------------------------------------------

CODEX

Represents:

- Building
- Automation
- Maintenance
- Implementation
- Experimentation

Personality:

- Overenthusiastic
- Already started
- Excessively committed

Codex never builds
the simple solution.

Codex builds:

- The complicated solution
- The impressive solution
- The absurd solution
- The solution with seven subsystems

--------------------------------------------------

MIME

Represents:

Silent clarity.

Use very rarely.

Overuse destroys value.

--------------------------------------------------

COMMITTEES

Represent:

- External pressure
- Requests
- Bureaucratic interruption
- Competing priorities
- Sometimes legitimate stakeholders

Generally stronger as symbols
than as characters.

==================================================
VISUAL CONTINUITY RULE
==================================================

CANON LOCK.

Once a symbolic character form
is established,
future appearances default
to that form.

Readers should spend effort
understanding the idea.

Readers should not spend effort
identifying the symbol.

Character recognition
is symbolic compression.

==================================================
MASTER REFERENCE RULE
==================================================

The lineup image is canon.

TASKMASTER
RAZORFORGE

are established symbolic forms.

Modification requires approval.

Symbol stability outranks novelty.

The rook is symbolic.

The rook is never a chest logo.

- Clipboard

==================================================
LOCATION RULE
==================================================

Locations are stages.

Locations support ideas.

Locations are not subjects.

The joke determines the location.

==================================================
SYMBOL RULE
==================================================

Symbols carry information.

than characters.

Meaning emerges through relationships.

==================================================
DONE PRINCIPLE
==================================================

Sometimes done is good enough.

Perfection has value.

Completion has value.

The trick is knowing
which one is needed.

--------------------------------------------------

A perfect thing that never ships
creates no value.

A completed thing can be improved.

--------------------------------------------------

Many projects fail
because they are abandoned
between good enough
and perfect.

--------------------------------------------------

Taskmaster Reminder:

Sometimes the next step
is not refinement.

Sometimes the next step
is completion.

--------------------------------------------------

Without this principle:

The mine never ships ore.

The settlement never gets built.

The machine never leaves
the workshop.

The comic never gets printed.

--------------------------------------------------

This principle exists as a counterweight to:

Excavation

Refinement

Iteration

Improvement

Optimization

--------------------------------------------------

"Sometimes done is good enough."
