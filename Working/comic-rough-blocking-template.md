# Comic Rough Blocking Template

Status: reusable working template

Purpose:

Use this format as the first draft for future multi-panel comic ideas after Taskmaster and the assistant have discussed and chosen a direction.

This is for testing movement, scale, character placement, and sequence logic before investing in polished rendering.

## When To Use

Use for:

- new multi-panel comic concepts
- story movement tests
- panel-by-panel staging
- rough page rhythm
- character/object placement
- visual gag timing

Do not use polished full-render image generation as the first blocking pass unless Taskmaster specifically asks for it.

## Output Level

Keep the first draft low-detail:

- simple shapes for characters
- readable silhouettes
- rough objects
- labels allowed only for internal blocking clarity
- no final rendering
- no texture work
- no detailed lighting
- no final perspective pressure

## Standard Six-Panel Blocking Fields

For each panel, define:

- panel number
- panel job
- rough composition
- key shapes / characters
- scale relationship
- movement from previous panel
- setup or payoff role

## Communication Rule

When direction has already been chosen, create and store the blocking artifact without surfacing prompt text or process scaffolding unless Taskmaster asks for it.

Continue working until a true Taskmaster decision is needed.

## Visual Development Guardrail

After rough blocking, do not jump into realistic or highly rendered panel art by default.

For this comic project, visual development should preserve the established comic-strip / animation language:

- simple readable character shapes
- plain backgrounds unless story-critical
- black-and-white line art as the default
- restrained grayscale
- sparse props
- character designs anchored to the lineup reference
- no photorealistic rendering
- no painterly realism
- no cinematic lighting
- no dense background clutter

Before rendering a panel with Taskmaster, Razorforge, or Codex, check the lineup reference:
