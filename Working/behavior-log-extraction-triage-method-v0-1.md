## Purpose

The behavior log is source material, not a single destination.

This method separates observations before interpretation so behavior-log material is not accidentally forced into comic canon, behavior profile, campaign system design, workplace commentary, or private meaning when it belongs somewhere else.

The governing question is:

How do observations survive long enough to become meaning without being frozen, flattened, or misplaced?

## Triage Lanes

### 1. Comic Source

Use for observations that clarify:

- Taskmaster as comic role
- Razorforge as interpretation/refinement role
- Codex as implementation/overbuild role
- committees, review spiral, fragile details, noodles, symbols, mythology, visual language, or Around Here
- future comic scripts, panels, image prompts, dictionary pages, or character notes

Do not promote to canon automatically.

Default destination:

`C:\comics\Source\References\`

or, for unresolved concept work:

`C:\comics\Working\`

### 2. Behavior Profile

Use for observations about:

- how Taskmaster works, decides, focuses, defers, revises, or stops
- overload, recovery, friction, persistence, curiosity, or pacing
- interaction preferences with AI
- recurring decision patterns
- what support or response style helps Taskmaster think clearly

Do not turn this into comic canon unless Taskmaster separately approves the comic use.

Default destination:

Behavior discussion/profile repository or export update file, not the comic canon folder.

### 3. Workflow Methodology

Use for reusable process doctrine:

- no micro-approval loops
- human-facing controls
- single intake / reuse everywhere
- source vs derivation
- draft vs canon
- approval gates
- no-write boundaries
- migration checks
- output usability

This lane can inform comic symbols, but its primary destination is methodology and process design.

Default destination:

Project methodology notes, campaign-development doctrine, or comic source references if the idea is being adapted visually.

### 4. Campaign-System Material

Use for Grayline or campaign-development mechanics:

- repository workflows
- relationship ledger
- supporting materials
- reports and table outputs
- registry behavior
- session extraction
- campaign canon or draft candidates

Do not store campaign authority inside the comic project except as extracted comic source material.

Default destination:

`C:\CAMPAIGN_PROJECT_DEVELOPMENT`

or the active campaign repository/write-gated destination, as governed by that project.

### 5. Workplace / Real-World Systems Context

Use for observations grounded in:

- executive director role
- human-services system responsibility
- oversight, training, standards, provider networks, or human burden
- office AI commentary
- professional communication patterns

This material can inform Taskmaster's character and occasional office strips, but should not turn the main comic into literal workplace administration unless Taskmaster asks.

Default destination:

Comic source reference or separate workplace context note.

### 6. Private / Park-Only

Use for observations that may matter for self-understanding but should not be reused broadly.

Examples:

- sensitive personal material
- material that is emotionally useful but not project-usable
- observations too fragile to classify
- material that should not become humor, canon, doctrine, or workflow law

Default action:

Park, summarize minimally, or leave unextracted unless Taskmaster explicitly approves preservation.

## Extraction Card Format

Use one card per observation:

```text
Observation:
[short source-grounded observation]

Source context:
[where it came from / what kind of behavior-log moment it was]

Meta-question:
[meaning formation / preservation / classification / framework / drift / translation / map / symbol / humor / Taskmaster / extraction / boundary]

Triage lane:
[comic source / behavior profile / workflow methodology / campaign-system material / workplace context / private-park-only]

Confidence:
[strong / tentative / fragile]

Use:
[what this can safely inform]

Do not use for:
[what this should not become]

Destination:
[suggested folder, repository, export update, or parked status]

Canon status:
[not canon / candidate / approved only after Taskmaster decision]
```

## Meta-Question Routing

When an observation is hard to classify, ask which engine question it belongs under:

- Meaning formation: How does meaning form?
- Preservation: How do we preserve meaning without freezing it?
- Classification: What happens when something does not fit?
- Framework: What framework is missing?
- Drift: How does meaning get lost?
- Translation: What survives migration?
- Map: Where are the relationships?
- Symbol: When does a joke become a symbol?
- Humor: What is funny Around Here?
- Taskmaster: What is Taskmaster actually doing?
- Extraction: How much context can be extracted without destroying emergence?
- Boundary: What is not Around Here?

If none fits, classify as boundary, private-park-only, or unresolved.

## Safe First Pass

For a first behavior-log pass:

1. Preserve source location and date.
2. Extract observations without interpretation inflation.
3. Assign one primary triage lane.
4. Assign one secondary lane only if genuinely useful.
5. Add the governing meta-question.
6. Mark confidence.
7. State what the observation must not be used for.
8. Recommend a destination.
9. Do not write canon, campaign repository files, or behavior profile conclusions without the relevant approval boundary.

## Decision Boundary

Stop and ask Taskmaster when:

- an observation is sensitive and might be private/park-only
- a comic use would turn behavior-profile material into a joke
- campaign canon or repository writes would be affected
- a behavior-profile conclusion would be stated as stable
- a new canon term, symbol, or character rule would be created
- the extraction reveals the need for a missing boundary question
