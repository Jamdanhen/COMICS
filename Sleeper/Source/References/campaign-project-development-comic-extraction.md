This document extracts comic-usable process language from the campaign development project. It is not campaign canon. It is reusable symbolic and procedural material for the comic project.

## Core Transferable Premise

The campaign project is a machine for turning messy information into durable knowledge.

Comic translation:

Raw material enters as logs, notes, fragments, dictation, contradictions, and uncertain details.

Razorforge extracts structure, relationships, meaning, evidence, and usable decisions.

Codex builds machinery that moves the material through staging, review, approval, archive, repository, and derived outputs. The machinery often works, but tends to become visually elaborate.

Taskmaster decides what becomes authoritative and what remains staged, uncertain, deferred, or rejected.

## Repository Authority

Source principle:

The repository is permanent. Chats, interfaces, applications, and AI layers are temporary.

Comic translation:

The Archive is the durable vault. Everything else is scaffolding, smoke, rails, levers, lamps, temporary signs, or replaceable machinery.

Useful visual:

- A stone archive or vault surrounded by temporary mechanical arms.
- Chat scrolls that dissolve unless copied into archive drawers.
- Interface masks hanging from hooks while the Archive remains unchanged.

## Single Entry / Reuse Everywhere

Source principle:

Information should be entered once, reconciled once, stored once, and reused everywhere.

Comic translation:

One ore cart should feed many machines. If Taskmaster has to shovel the same ore into five machines, the machine has failed.

Useful visual:

- One central intake chute feeding multiple derived outputs.
- A warning sign: "Manual Duplication Detected."
- Codex proudly presenting five redundant input funnels while Razorforge circles the one real intake chute.

## Anti-Redundancy

Source principle:

Derive before duplicate. Consolidate before expand. Reference before recreate.

Comic translation:

Before Codex builds a new machine, Razorforge checks whether the machine already exists, whether the new machine duplicates an old machine, and whether it creates extra maintenance.

Useful visual:

- Razorforge holding a checklist in front of Codex's half-built second copy of the same contraption.
- Identical machines labeled "analysis," "re-analysis," "re-re-analysis."

## Evidence Before Canon

Source principle:

Information becomes authoritative only when supported by evidence, observation, confirmation, prior authority, or explicit approval.

Comic translation:

No object enters the Archive without a source tag, witness mark, approval stamp, or evidence tether.

Useful visual:

- A bouncer at the Archive door turning away a shiny but unlabeled artifact.
- Uncertain facts waiting in a staging pen with tags: "speculation," "deferred," "needs source."

## Fragile Detail Preservation

Source principle:

Fragile details should be preserved when useful for recall, validation, continuity, extraction, or reconstruction, without turning every uncertain phrase into a blocking decision.

Comic translation:

Small details are placed in padded boxes or tiny labeled jars. They are not canon jewels yet, but they are not thrown away.

Useful visual:

- Codex nearly sweeping "Destination unknown" into the furnace.
- Razorforge catches it with tongs and places it in a small "Preserved, Non-Blocking" jar.
- Taskmaster nods because no immediate decision is required.

## Anti-Spiral

Source principle:

Review exists to support decisions. A review step must not automatically create another review step.

Comic translation:

The Review Spiral is a dangerous machine that manufactures more review tickets from every review ticket. It must be stopped by choosing: accept, direct revision, or stop.

Useful visual:

- A corkscrew conveyor labeled "review" feeding itself.
- Taskmaster pulling a brake lever labeled "smallest useful next action."
- Razorforge replacing a stack of worksheets with one decision card.

## Human-Facing Workflow

Source principle:

Every human-facing step must be understandable, readable, and usable. The human should not have to do machine work.

Comic translation:

If Taskmaster has to copy machine-code labels, raw file paths, or multi-option blocks by hand, Codex has built the interface backward.

Useful visual:

- Codex unveils a control panel made entirely of tiny path labels.
- Razorforge covers it with plain decision buttons: approve, reject, revise, defer, stop.
- Taskmaster points at the one obvious next action.

## Approval Boundaries

Source principle:

Approval packets can approve proposed changes for later phases without authorizing durable repository writes.

Comic translation:

There are stamps with different powers. "Approved for review" is not the same stamp as "Write to Archive."

Useful visual:

- Codex reaching for the big "WRITE" lever.
- Taskmaster holding up a smaller stamp: "Approved for future write-capable phase only."
- Razorforge labels the boundary line.

## Phase Gates And Waypoints

Source principle:

The project advances through phases and waypoints. Completed phases should not restart without cause.

Comic translation:

Waypoints are gates in the mine/forge/archive route. Passing a gate means the cart moves forward. It does not circle back unless a contradiction, new evidence, design failure, or explicit order appears.

Useful visual:

- Track switches labeled Phase 5, Phase 6, Phase 9B, Phase 9C, Phase 10.
- A sign: "Do not reopen completed gate without cause."

## Comic-Usable Roles

Taskmaster:

- Owns the decision.
- Protects continuity.
- Chooses when to proceed, revise, defer, or stop.

Razorforge:

- Extracts the governing principle.
- Prevents needless duplication.
- Preserves fragile details without inflating them into blockers.
- Converts machine-facing complexity into human-facing clarity.

Codex:

- Builds the extraction machine, approval interface, proposal generator, staging archive, and supporting contraptions.
- Often makes the machinery technically effective before it is pleasant to use.
- May need Razorforge and Taskmaster to reduce path-heavy, overbuilt machinery into readable decisions.
