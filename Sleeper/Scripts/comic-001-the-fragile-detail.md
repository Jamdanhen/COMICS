## Principle

Useful uncertain details should be preserved without being promoted to canon or turned into an immediate blocking decision.

## Why This Principle

This principle is valuable because it protects memory without creating unnecessary work. It is also visually adorable: a tiny uncertain scrap can be saved in a careful little jar instead of being burned, ignored, or inflated into a giant decision.

## Panel Logic

Panel 1:

Taskmaster presents a tiny uncertain detail: `destination unknown`.

Panel 2:

Codex eagerly overbuilds the problem into a machine with two excessive options: make it canon or burn it.

Panel 3:

Razorforge extracts the actual principle: preserve the detail without over-deciding it.

Panel 4:

The detail is safe in the Fragile Detail Jar. Taskmaster approves. Codex still adds one unnecessary steam whistle because Codex is Codex.

## Symbols Used

- Taskmaster: authority, continuity, ownership.
- Razorforge: extraction, refinement, relationship clarity.
- Codex: eager implementation, overbuilt machinery, useful excess.
- Fragile Detail Jar: preservation without canonization.
- Make Canon lever: premature authority.
- Burn It chute: premature discard.
- Steam whistle: unnecessary but satisfying Codex flourish.
